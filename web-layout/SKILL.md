# Web Layout — 공통 레이아웃 구현 스킬

이 스킬의 목적: 모든 페이지에서 공유하는 **레이아웃 뼈대**를 만든다.
레이아웃이 잘 잡혀야 이후 페이지 구현이 빠르고 일관적이다.

---

## 진행 순서

```
Step 1: 기존 설정 확인 (설계서, CLAUDE.md)
Step 2: 레이아웃 패턴 확정
Step 3: 컴포넌트 순서대로 구현
Step 4: 반응형 처리
Step 5: 동작 확인
```

---

## Step 1: 기존 설정 확인

구현 전에 아래 파일들을 확인한다:

1. **설계서** → 네비게이션 구조, 페이지 목록, 사용자 역할
2. **CLAUDE.md** → 디자인 규칙 (컬러, 간격, 컴포넌트 스타일)
3. **src/lib/constants.ts** → 메뉴 항목이 이미 정의되어 있는지

없으면 사용자에게 최소한 아래를 확인한다:
- 사이드바 메뉴 항목 (몇 개, 이름, 아이콘)
- 헤더에 뭘 넣을지 (로고, 사용자 정보, 검색 등)
- 다크모드 필요 여부

---

## Step 2: 레이아웃 패턴 확정

사내 관리 도구에서 가장 일반적인 패턴은 **사이드바 고정형**이다.

### 패턴 A: 사이드바 고정 (기본 추천)

```
┌──────────────────────────────────────┐
│ 사이드바(256px)│ 헤더                 │
│               │────────────────────── │
│  로고          │                      │
│  메뉴1        │    컨텐츠 영역        │
│  메뉴2        │                      │
│  메뉴3        │                      │
│               │                      │
│  ──────       │                      │
│  설정          │                      │
│  로그아웃      │                      │
└──────────────────────────────────────┘
```

### 컴포넌트 분리 구조

```
src/components/layout/
├── RootLayout.tsx       # 전체를 감싸는 최상위 레이아웃
├── Sidebar.tsx          # 사이드바 (메뉴, 로고)
├── Header.tsx           # 상단 바 (검색, 사용자 정보, 알림)
├── Footer.tsx           # 하단 (선택)
└── MobileMenu.tsx       # 모바일 햄버거 메뉴
```

---

## Step 3: 컴포넌트 순서대로 구현

반드시 아래 순서로 하나씩 만든다. 한 번에 전부 만들지 않는다.

### 3-1. RootLayout 먼저

전체 구조를 잡는 컨테이너. 사이드바와 컨텐츠 영역의 배치만 담당한다.

포함할 것:
- 사이드바 영역 (고정 폭)
- 컨텐츠 영역 (나머지 폭)
- 헤더 영역
- children을 받아서 컨텐츠 영역에 렌더링

주의:
- 로그인 페이지는 이 레이아웃을 사용하지 않는다 (별도 레이아웃)
- 라우터에서 레이아웃 적용/미적용 분기 처리 필요

### 3-2. Sidebar

가장 중요한 컴포넌트. 구현 시 반드시 포함할 것:

**필수:**
- 상단 로고/서비스명
- 메뉴 항목 (아이콘 + 텍스트)
- 현재 페이지 하이라이트 (active 상태)
- 메뉴 클릭 시 해당 페이지로 이동

**권장:**
- 메뉴 그룹 구분선 (주요 메뉴 / 설정 메뉴)
- 접이식 서브메뉴 (하위 메뉴가 있는 경우)
- 하단에 사용자 정보 또는 로그아웃

**메뉴 데이터 관리:**
메뉴 항목은 하드코딩하지 말고 constants.ts에서 가져온다:

```typescript
// src/lib/constants.ts
export const MENU_ITEMS = [
  { 
    label: '대시보드', 
    path: '/dashboard', 
    icon: 'LayoutDashboard' 
  },
  {
    label: '거래 관리',
    icon: 'FileText',
    children: [
      { label: '거래 목록', path: '/trades' },
      { label: '거래 등록', path: '/trades/new' },
    ]
  },
];
```

### 3-3. Header

사이드바 오른쪽 상단에 위치하는 바.

**포함할 것 (상황에 따라 선택):**
- 현재 페이지 제목 또는 breadcrumb
- 검색바 (전체 검색 필요 시)
- 알림 아이콘 (알림 기능 있을 때)
- 사용자 프로필 (이름, 아바타, 드롭다운 메뉴)
- 모바일 햄버거 메뉴 버튼

### 3-4. MobileMenu (반응형)

모바일에서 사이드바를 대체하는 메뉴.

구현 방식:
- md 이하에서 사이드바 숨김
- 햄버거 버튼 클릭 → 오버레이 + 슬라이드 메뉴
- 메뉴 항목은 Sidebar와 동일한 데이터(MENU_ITEMS) 사용
- 메뉴 외부 클릭 또는 항목 선택 시 닫힘

---

## Step 4: 반응형 처리

```
lg (1024px+):  사이드바 표시 + 전체 레이아웃
md (768px):    사이드바 접힘 (아이콘만) 또는 숨김
sm (640px-):   사이드바 숨김 + 햄버거 메뉴
```

### 반응형 체크리스트

```
□ 사이드바가 모바일에서 올바르게 숨겨지는가
□ 햄버거 메뉴가 정상 동작하는가 (열기/닫기)
□ 컨텐츠 영역이 사이드바 없을 때 전체 폭을 사용하는가
□ 헤더가 모바일에서도 잘 보이는가
□ 터치 대상이 44px 이상인가
```

---

## Step 5: 동작 확인

```
□ 메뉴 항목 클릭 시 페이지 이동이 되는가
□ 현재 페이지에 해당하는 메뉴가 하이라이트되는가
□ 로그인 페이지에서는 사이드바가 나타나지 않는가
□ 컨텐츠 영역 스크롤 시 사이드바/헤더가 고정되어 있는가
□ 사이드바 메뉴가 많을 때 사이드바 내부 스크롤이 되는가
□ 브라우저 크기를 줄였을 때 반응형이 자연스러운가
```

확인 후 사용자에게 결과를 보여주고, 수정 요청을 받는다.
레이아웃이 확정되면 다음 단계(페이지 구현)로 넘어간다.

---

## 주의사항

- 레이아웃은 한 번 잡으면 모든 페이지에 영향을 주므로 이 단계에서 꼼꼼히 잡는다
- 디자인 컬러와 간격은 CLAUDE.md에 정의된 값을 반드시 사용한다
- 아이콘 라이브러리는 프로젝트 전체에서 하나만 사용한다 (lucide-react 추천)
- 레이아웃 컴포넌트에 비즈니스 로직을 넣지 않는다 (순수하게 구조만 담당)
- 메뉴 데이터는 constants.ts에서 관리하여 수정이 쉽도록 한다

---

## 레이아웃 문구/코드 품질 규칙

레이아웃 컴포넌트에서도 CLAUDE.md의 품질 규칙을 반드시 적용한다.

### 메뉴/헤더 텍스트

```
메뉴 라벨: 명사형, 간결하게
  ✅ "대시보드", "거래 관리", "설정"
  ❌ "대시보드 보기", "거래를 관리하기", "설정하기"

서브메뉴: 상위 메뉴와 중복 단어 제거
  ✅ 거래 관리 > "목록", "등록"
  ❌ 거래 관리 > "거래 목록", "거래 등록"

헤더 영역: 로그아웃 버튼은 "로그아웃" 통일 ("나가기", "종료" ❌)
```

### 레이아웃 코드 품질

```
□ 레이아웃 컴포넌트 변수명이 명확한가 (isSidebarOpen, menuItems 등)
□ 메뉴 항목이 하드코딩되지 않고 constants.ts에서 가져오는가
□ 반응형 브레이크포인트가 Tailwind 기본값과 일치하는가
□ 컴포넌트 props 타입이 정의되어 있는가
□ 주석이 한국어로 작성되어 있는가
```
